# Mock API 사용 가이드

## 개요

백엔드 서버가 없을 때도 프론트엔드를 테스트할 수 있도록 Mock API 모드를 제공합니다.

## Mock 모드 활성화 방법

**기본 설정**: 개발 환경에서는 **기본적으로 Mock 모드가 활성화**되어 있습니다!

따라서 별도의 설정 없이도 바로 사용할 수 있습니다. 

만약 실제 백엔드 서버를 사용하고 싶다면, 프로젝트 루트에 `.env` 파일을 생성하고 다음을 추가하세요:

```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL=http://localhost:8080/api
```

**중요**: 
- `.env` 파일을 생성하거나 수정한 후에는 **개발 서버를 재시작**해야 합니다!
- 브라우저 콘솔에서 "🔧 개발 모드" 메시지를 확인하면 Mock 모드가 활성화되었는지 확인할 수 있습니다.

- ✅ 비밀번호 재설정
- ✅ 토큰 갱신

### 사용자 관리

- ✅ 프로필 수정
- ✅ 비밀번호 변경
- ✅ 알림 설정 변경
- ✅ 회원 탈퇴

### 일기 관리

## 테스트 계정

Mock 모드에서는 다음 테스트 계정을 사용할 수 있습니다:

- **이메일/아이디**: `test@example.com`
- **비밀번호**: `password123`

## Mock 데이터 저장

Mock 모드는 브라우저의 메모리에 데이터를 저장합니다. 따라서:
- 페이지를 새로고침하면 작성한 일기는 유지됩니다
- 브라우저를 완전히 닫으면 데이터가 초기화됩니다
- 다른 브라우저나 시크릿 모드에서는 별도의 데이터를 사용합니다

## 실제 백엔드로 전환

백엔드 서버가 준비되면 `.env` 파일에서 다음을 변경하세요:

```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL=http://localhost:8080/api
```

또는 `VITE_USE_MOCK_API`를 제거하면 자동으로 실제 API를 사용합니다.

## 주의 사항

1. Mock 모드는 개발 및 테스트 목적으로만 사용하세요
2. Mock 데이터는 실제 데이터베이스와 구조가 다를 수 있습니다
3. 일기 내용의 암호화는 Mock 모드에서 시뮬레이션되지 않습니다
